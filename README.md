# UI-Sass-landingPage
A modern, responsive SaaS landing page built with HTML, Sass, and vanilla JavaScript.  
This project is focused on clean UI, reusable SCSS architecture, and production-ready front-end structure.
🔗 **Live Demo:** [UI-Sass-landingPage on GitHub Pages](https://mansour-vn.github.io/UI-Sass-landingPage/)
---
## Features
- **Modern SaaS landing layout**  
  Hero section, features, stats, testimonials, CTA, blog/news, and footer.
- **Responsive design**  
  Optimized for desktop, tablet, and mobile screen sizes.
- **Sass-based styling**  
  Modular SCSS files, variables, mixins, and partials for easier maintenance.
- **Reusable UI components**  
  Buttons, cards, sections (features, testimonials, events, news) designed to be extended.
- **Clean, semantic HTML**  
  Accessible structure for better SEO and screen readers.
---
## Sections Overview
- **Hero / Intro** – Main headline, description, primary and secondary call-to-action buttons.  
- **Business Solutions** – Cards describing key product/service features.  
- **Key Metrics** – Download counts, active users, and providers.  
- **Content / Highlights** – Informative sections with titles and supporting text.  
- **Benefits List** – Multiple bullet points outlining advantages and use cases.  
- **Testimonials** – Customer feedback with name, role, and quotes.  
- **Call to Action** – “What are you waiting for?” section with a strong CTA button.  
- **Events & News** – Short cards for blog posts, announcements, or updates.  
- **Footer** – Company links, brief description, language switcher.

## Getting Started
### 1. Clone the repository
git clone https://github.com/<your-username>/UI-Sass-landingPage.git
cd UI-Sass-landingPage
2. Install dependencies (if you use a build tool)
If you have a package.json (for Sass compilation or tooling):

npm install
3. Run Sass (example)
If you use the Sass CLI:

sass --watch src/scss:dist/css
Adjust the paths above to match your folder structure.

4. Open in browser
Simply open the index.html file in your browser, or use a local server (for example, VS Code Live Server).

Development
Organize styles into logical SCSS partials (e.g. base, layout, components, pages).
Use variables for colors, spacing, typography to keep the design consistent.
Keep components small and reusable where possible.
Deployment
This project is deployed using GitHub Pages:

Branch: typically main or master
Build: static files (HTML, CSS, JS)
URL:
https://mansour-vn.github.io/UI-Sass-landingPage/
